# Do czego służą dane foldery

Kod jest pisany w `/lib`. W środku mamy strukturę:

1. **models/** - modele danych, czyli klasy z danymi
2. **providers/** - zarządzanie stanem (firebase i auth providers)
3. **screens/** - widoki aplikacji
4. **services/** - logika biznesowa, czyli API, firebase itd.
5. **ui/** - podstawowe komponenty z których korzystamy przy tworzeniu UI
6. **ui/theme/** - styl aplikacji
6. **ui/widgets/** - większe komponenty z logiką (widgety)
7. **main.dart** - punkt wejścia aplikacji
8. **app.dart** - główna konfiguracja aplikacji
9. **routes.dart** - konfiguracja nawigacji
